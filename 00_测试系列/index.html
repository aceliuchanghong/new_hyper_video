// 第 01 节 · 定义
什么是 Agent?
在目标驱动下，自主感知、决策、行动的闭环系统。
model · harness · loop
// 定义
CORE DEFINITION
Agent 是一个能在环境中根据目标自主选择行动，并根据行动反馈继续调整下一步行为的闭环系统。
// 一句话工程定义
Agent = Model + Harness + Loop
Model负责判断下一步做什么
Harness让模型能看见环境、调用工具、执行动作
Loop把反馈送回模型，直到任务完成
// 常见误解
流程写死的，是 workflow，
不是 Agent。
硬编码流水线
用户输入 → 分类 → 固定分支 → 固定工具 → 固定格式输出
核心决策来自代码，而非模型在环境反馈中自主选择。
// Agent 的身体
Harness = 模型能行动的环境
Tools文件 · Shell · 网络 · 浏览器
Knowledge文档 · API 规范 · 风格指南
Observationgit diff · 日志 · 状态
ActionCLI · API · UI 交互
Permissions沙箱 · 审批 · 信任边界
模型提供智能，Harness 提供身体。
